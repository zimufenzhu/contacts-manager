# Android真机测试计划

## 1. 测试环境准备

### 1.1 设备要求
- Android 5.0及以上版本的手机或平板电脑
- 至少1GB RAM
- 至少100MB可用存储空间

### 1.2 测试账号准备
- 准备至少3个测试联系人数据
- 包含各种字段类型的联系人信息

### 3.1 不同Android版本测试

- [ ] Android 7.0-8.0
- [ ] Android 9.0-10.0
- [ ] Android 11.0及以上

### 3.2 不同屏幕尺寸测试
- [ ] 小屏手机（4-5英寸）
- [ ] 大屏手机（6-7英寸）
- [ ] 平板电脑（8英寸及以上）

## 4. 性能测试

### 4.1 启动性能
- [ ] 冷启动时间 < 3秒
- [ ] 热启动时间 < 1秒

### 4.2 操作响应性
- [ ] 界面切换流畅
- [ ] 数据加载及时
- [ ] 无明显卡顿现象

## 5. 稳定性测试

### 5.1 长时间运行测试
- [ ] 连续使用1小时无崩溃
- [ ] 多次切换界面无异常

### 5.2 异常情况处理
- [ ] 网络中断时的表现
- [ ] 存储空间不足时的表现
- [ ] 低内存情况下的表现

### 9.3 优化后的文件结构
```
├── main.py                 # 应用入口文件
├── models.py               # 数据模型定义
├── buildozer.spec          # Android打包配置
├── requirements.txt        # 项目依赖
├── README.md               # 项目说明文档
├── RELEASE_NOTES.md        # 发布说明文档
├── ANDROID_TESTING_PLAN.md # 测试计划文档
├── DEVELOPMENT_DOC.md      # 开发文档
├── USER_MANUAL.md          # 用户手册
├── CONSOLIDATED_DOCS.md    # 整合文档
├── build_apk.bat           # Docker构建脚本
├── test_core_functions.py  # 核心功能测试
├── database/
│   ├── __init__.py
│   └── db_manager.py       # 数据库管理器
├── utils/
│   ├── __init__.py
│   ├── contact_manager.py  # 联系人管理
│   ├── field_manager.py    # 自定义字段管理
│   ├── search_manager.py   # 搜索功能
│   └── export_manager.py   # 数据导出功能
└── screens/
    ├── __init__.py
    ├── main_screen.py      # 主界面
    ├── contact_list_screen.py  # 联系人列表界面
    ├── contact_detail_screen.py # 联系人详情界面
    ├── contact_edit_screen.py  # 联系人编辑界面
    └── settings_screen.py      # 设置界面
```
